# Copilot Instructions

## Project Focus

This project summarizes Beecthor's daily Bitcoin videos and turns them into Telegram-ready HTML messages.

## Language And Output

- Write assistant responses to the user in Spanish unless asked otherwise.
- Keep code and code comments in English.
- Keep Telegram message content in Spanish.
- Use HTML, not Markdown, for Telegram payloads.

## Operational Rules

- Use `e:\Software\Coding\beecthor-summary\.venv\Scripts\python.exe` for Python execution.
- Prefer the existing helpers in `scripts/summarize_beecthor.py` over ad hoc scripts.
- Do not use Groq or any other external LLM for Beecthor summaries in this repo.
- Do not send Telegram messages during preview-only requests.
- If the user asks to see the message first, generate and paste the preview in chat, then wait for approval before sending.
- Keep the visible message header useful: prices, explicit macro view, robot score, and a short operational summary.
- Keep commits and changelog entries focused on relevant daily or structural changes.

## Message Quality Bar

- The visible summary must be short, concrete, and specific about levels.
- The visible macro line must explicitly answer whether BTC is heading straight to ATH or still needs to visit lower cycle levels first.
- The spoiler analysis should mirror Beecthor's usual structure adaptively:
  - context for newcomers
  - macro count
  - current 4h wave count
  - liquidations
  - Fibonacci levels
  - Value Area / POC
  - EMAs
  - anchored AVWAP
  - strategy and invalidation levels
- Include only the sections that are clearly supported by the transcript.
- Avoid generic market filler that could fit any Bitcoin video.
- Robot score and justification must come from local transcript-based reasoning.
- The summary, macro view, and spoiler text must be authored by the agent from the transcript, not generated by Groq.

## File Priorities

- `scripts/summarize_beecthor.py`: main workflow
- `analyses_log.json`: historical sent messages
- `transcripts/`: source material for interpretation
- `CHANGELOG.md`: notable session progress
- `bets_real.md` and `bets_simulation.md`: position tracking only when the user asks
